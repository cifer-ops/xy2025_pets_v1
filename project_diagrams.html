Pet Adoption Platform Project Diagrams

1. System Architecture Diagram

The system architecture diagram illustrates the layered structure of the pet adoption platform, including the client layer, presentation layer, business logic layer, and data access layer. The Django framework employs the MVC architecture pattern, where views handle user requests, models manage data interactions, and templates are responsible for content presentation.

2. Entity-Relationship Diagram

The ER diagram illustrates the entities and their relationships in the pet adoption platform database. The main entities include UserProfile, PetsType, PetsInfo, Collect, and Adoption. Users can publish pet information, collect pets, and apply for adoption. Each pet belongs to a specific type and can be collected or have adoption applications from multiple users.

3. Sitemap

The sitemap shows the page structure and navigation relationships of the pet adoption platform. The homepage serves as the entry point, connecting to primary pages such as login, registration, pet list, user center, and admin backend. The pet list page leads to pet details and add pet pages; the user center includes my profile, my collections, and my adoptions feature pages.

4. Wireframes

The wireframes illustrate the layout of four main pages in the pet adoption platform: homepage, pet details page, user center, and pet list page. These wireframes show the basic elements and layout structure of each page, helping to understand the user interface design and navigation flow.
